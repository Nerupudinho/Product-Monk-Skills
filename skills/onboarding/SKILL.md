## 1. The PM Framework

All work flows through this hierarchy:

```
Vision
  └── Goals  (quarterly / half-yearly OKRs)
        └── Strategy  (living doc — reviewed daily)
              └── Initiatives  (concrete work items)
                    └── Roadmap  (backlog → in progress → done)
```

**Key principle:** Goals don't change within the quarter. Strategy changes constantly — it adapts to new information, blockers, and environment shifts. The strategy doc is the most alive document in the system.

Full framework doc: `Company Level Context/Processes/PM Framework.md`

## 4. Daily Operating Rhythm

| Command | What it does |
|---------|-------------|
| `/plan-day` | Fetch calendar + categorise tasks (Overdue / Due Today / In Progress) + generate `this-week.md` and `next-week.md` |
| `/prioritisation` | Go through task list, collect status updates, write updated task file |
| `/log-meeting` | Paste a transcript → structured meeting file + update initiative docs |
| `/funnel-update` | Format raw Repeat funnel numbers into Slack-ready daily update |
| `/meeting-prep` | Prep doc for an upcoming meeting — context, alignment questions, asks |
| `/strategic-build` | Evaluate whether something is worth building (LNO + Pre-Mortem) |
| `/update-action-items` | Update action items in a recently logged meeting file |
| `/update-open-items` | Update `Open Items.md` with status changes or new items |

## 6. Task File Format

Daily task files (`Tasks/YYYY-MM-DD.md`) have three sections:
- **Overdue** — tasks with a past deferred date not yet done
- **Due Today** — in-progress today + deferred-to-today + top-priority actions
- **In Progress Initiatives** — ongoing work, monitoring, future-deferred items

Weekly files generated by `/plan-day`:
- `Tasks/this-week.md` — day-by-day view (meetings + tasks per day)
- `Tasks/next-week.md` — confirmed next-week deferrals + calendar preview

## 7. Core Rules

- **Reference before writing.** Find 2–3 existing examples and match their structure.
- **Full file paths always.** Never refer to a file by name alone.
- **Ask when unsure.** Never invent table names, schema, or placeholder content.
- **Strategy is alive.** After any significant event or decision, ask: does the strategy doc need updating?
